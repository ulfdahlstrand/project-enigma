# Backend Stack and API

## Tech Stack

| Concern | Technology | Notes |
|---------|-----------|-------|
| Runtime | **Node.js** | HTTP server hosting the oRPC handler |
| API framework | **oRPC** | End-to-end typed RPC; also exposes OpenAPI spec. **Not** tRPC. |
| Validation | **Zod** | Schemas shared via `@cv-tool/contracts` |
| Database client | **Kysely** | Type-safe SQL query builder for TypeScript. Uses the `pg` driver via PostgreSQL dialect. |

## oRPC

oRPC is the **sole** API communication layer between frontend and backend.

### Key Rules

1. No REST endpoints and no GraphQL.
2. Zod schemas define all procedure inputs and outputs in `@cv-tool/contracts`.
3. The backend implements the oRPC router.
4. The frontend imports the router type from `@cv-tool/contracts`.
5. Transport is HTTP/JSON via the oRPC handler.

## Adding a New Procedure

1. Define Zod schemas in `@cv-tool/contracts`.
2. Implement the handler in `apps/backend/`.
3. Consume it from `apps/frontend/`.

## OpenAPI

An OpenAPI specification is automatically generated by oRPC for documentation
and possible future non-TypeScript clients.

## Shared Contracts

The oRPC router type and all shared Zod schemas live exclusively in
`packages/contracts/`.

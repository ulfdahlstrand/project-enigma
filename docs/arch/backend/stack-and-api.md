# Backend Stack and API

## Tech Stack

| Concern | Technology | Notes |
|---------|-----------|-------|
| Runtime | **Node.js** | HTTP server hosting the oRPC handler |
| API framework | **oRPC** | End-to-end typed RPC; also exposes OpenAPI spec. **Not** tRPC. |
| Validation | **Zod** | Schemas shared via `@cv-tool/contracts` |
| Database client | **Kysely** | Type-safe SQL query builder for TypeScript. Uses the `pg` driver via PostgreSQL dialect. |

## oRPC

oRPC is the **sole** API communication layer between frontend and backend.

### Key Rules

1. All public app APIs are defined in `@cv-tool/contracts` with explicit HTTP method and path metadata.
2. Zod schemas define all procedure inputs and outputs in `@cv-tool/contracts`.
3. The backend implements the oRPC router and exposes it through `OpenAPIHandler`.
4. The frontend imports the router type from `@cv-tool/contracts` and uses `OpenAPILink`.
5. Reads should use `GET` where practical.
6. CRUD writes should use `POST`, `PATCH`, and `DELETE`.
7. Command-style operations such as exports, AI actions, and some versioning flows may stay explicit action routes.

## Adding a New Procedure

1. Define or update the Zod schemas in the relevant contract domain.
2. Add or update the procedure in `packages/contracts/src/domains/{domain}/routes.ts`.
3. Compose the domain route group in `packages/contracts/src/index.ts`.
4. Add an explicit `.route({ method, path })` unless there is a deliberate reason to keep it implicit.
5. Implement the handler in `apps/backend/`.
6. Consume it from `apps/frontend/`.

## OpenAPI

An OpenAPI specification is automatically generated by oRPC for documentation
and possible future non-TypeScript clients.

## API Shape

The backend now follows a resource-oriented HTTP shape on top of oRPC/OpenAPI.

Examples:

- `GET /employees/{id}`
- `GET /resumes/{id}`
- `POST /resume-branches/{branchId}/commits`
- `POST /ai/conversations/{conversationId}/messages`

The transport is still oRPC, but the externally visible API should read like a normal HTTP API instead of fallback procedure paths such as `/getEmployee`.

## Shared Contracts

The oRPC router type and all shared Zod schemas live exclusively in
`packages/contracts/`.

### Contract Layout

Contract route definitions are grouped by domain under:

- `packages/contracts/src/domains/{domain}/routes.ts`

The root contract file:

- `packages/contracts/src/index.ts`

is the composition point. It re-exports shared schemas and types, and combines
the domain route groups into the final `contract`.

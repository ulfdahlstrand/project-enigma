## Current conclusion

The existing in-repo MCP work was a useful validation path, but the likely long-term direction is:

- keep the MCP layer thin
- stop coupling it to frontend/dev proxy concerns more than necessary
- move toward a dedicated MCP service with connector-compatible auth/bootstrap
